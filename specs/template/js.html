Laconic JavaScript (LJS)

NFX Template Compiler Tool allows making special laconic-syntax inserts (we named that technology as Laconic Java Script - LJS) into javascript. These code segments present structure of HTML DOM elements in laconic format. The tool translates them into javascript statements of DOM elements building. To make ntc tool take into account LJS code you should specify certain compiler NFX.Templatization.TextJSTemplateCompiler or NFX.Templatization.NHTCompiler (for whole page):

ntc.exe src.js -ext ".result" /src /c "NFX.Templatization.TextJSTemplateCompiler, NFX"
ntc.exe my_page.nht -r -ext ".auto.cs" /src /c "NFX.Templatization.NHTCompiler, NFX"


LJS code is surrounded with /*** and ***/ bounds. In laconic format terms every DOM element becomes a segment and its parameters become attributes of that section, nested elements become sub-section, inner text of the element is transfered into value of the segment. You can specify which element will be root for the new tree - first argument of function with LJS (id or element itself). Other arguments also can be used in LJS code through ? js_code syntax but in this case the first argument of the function will be considered as a root even if you are not going specify it so you should add corresponding parameter.

function buildChangeSNameForm(root, dfltName) {
  /***
  form {
    id=frmChangeSName
    action="#"
    method=post
    div="Please enter new screen name:" {}
    input {
      type=text
      name=sname
      value=?dfltName
    }
    input {
      type=submit
      value=Submit
      class=uiSmallButton
    }
  }
  ***/
}

After compilation that LJS code turns into:

And somewhere in your code you can call this function: var form = buildChangeSNameForm(null, user.first_name).


More complicated than references to the arguments javascript statements (conditions, loops, etc.) should be defined as laconic section:

function withCondition(root, count) {
  /***
  "? var a=count*2;" {}
  input {
    id=limitExceeded
    type=checkbox
    checked="?(a>10)"
  }
  "?if (a>10)" {
    div {
      class="? a>100 ? 'divCritical' : 'divWarning'"
    }
  }
  ***/
}

After compilation we have:

You can use ?this as a pointer on current html element:

/***
table {
  "? buildRow(?this);" {}
}
***/

And we get:

var Ø1 = WAVE.ce('table');
buildRow(Ø1);


Also you can create pointer on any item in tree with construction ljsid=<your_identifier> and use it:

/***
div {
  ljsid=root
  div = "some text" {}
  "? if(root.hasChild())" {
     ...
   }
}
***/

Result:

var Ø1 = WAVE.ce('div');
var Ø2 = WAVE.ce('div');
Ø2.innerText = 'some text';
Ø1.appendChild(Ø2);

var div = "*#* div { h2='Header' {} p='Paragraph' {} } *#*";

Using of NHT code in LJS looks like:

/***
  span="?'?[:SomeVariable]'"
***/
